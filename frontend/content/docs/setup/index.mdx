DevBrain runs on Cognee, which supports two deployment modes:

- **Cognee Cloud:** hosted graph, vector, and relational stores. No local infra beyond the ingestion server. Recommended for most users.
- **Local mode:** file-based Kuzu graph + LanceDB vector + SQLite. LLM and embeddings via Gemini API. Zero cloud dependency.

Both approaches share the same prerequisites.

## Prerequisites

- **Python 3.11+** (for local setup)
- **Docker & Docker Compose** (for containerized setup)
- **Redis** (local setup only; Docker includes it)
- **Gemini API key** (default for local mode; get one at [aistudio.google.com/apikey](https://aistudio.google.com/apikey))
- **Cognee Cloud account** (optional; sign up at [cognee.ai](https://cognee.ai))
- **GitHub PAT** with `repo` read and webhook access

## Environment Variables

Copy `.env.example` to `.env` and configure the relevant variables for your chosen mode:

```bash tab="Cognee Cloud"
# Hosted stores - no local vector/graph infrastructure
COGNEE_BASE_URL=https://your-instance.cognee.ai
COGNEE_API_KEY=your_cognee_api_key

# GitHub access (required for both modes)
GITHUB_TOKEN=ghp_...
GITHUB_REPO=owner/your-repo
GITHUB_WEBHOOK_SECRET=...
```

```bash tab="Local Mode"
# File-based stores - Kuzu + LanceDB + SQLite, LLM via Gemini
GEMINI_API_KEY=...
LLM_PROVIDER=gemini
COGNEE_DATA_DIR=./.cognee/data
COGNEE_SYSTEM_DIR=./.cognee/system

# GitHub access (required for both modes)
GITHUB_TOKEN=ghp_...
GITHUB_REPO=owner/your-repo
GITHUB_WEBHOOK_SECRET=...
```
