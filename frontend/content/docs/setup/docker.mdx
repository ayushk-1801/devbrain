---
title: Docker Setup
description: Deploy DevBrain using Docker Compose.
---

Build and start all services with a single command:

```bash
docker-compose up --build
```

## Services

| Service | Port | Purpose |
|---------|------|---------|
| `redis` | internal | Job queue + changelog/profile state |
| `devbrain-api` | 8000 | FastAPI REST API + webhook receiver |
| `devbrain-worker` | - | ARQ worker - runs Cognee ingestion |
| `devbrain-mcp` | - | MCP server for Claude Code / Cursor (requires `--profile mcp`) |
| `devbrain-platform` | 9000 | Platform service for auth + instance provisioning (requires `--profile platform`) |

All Redis state and Cognee data persist across restarts via Docker volumes.

<Callout type="info" title="Cognee Mode">
DevBrain uses **Cognee Cloud** by default - set `COGNEE_API_KEY` and `COGNEE_BASE_URL` in your `.env`. Leave `COGNEE_API_KEY` blank to fall back to **local mode** (Kuzu + LanceDB + SQLite with Gemini). The backend auto-detects the mode.
</Callout>

## First Ingestion

```bash
curl -X POST "http://localhost:8000/ingest" \
  -H "Content-Type: application/json" \
  -d '{"repo": "owner/your-repo", "sync_history_days": 90}'
```

Check job status:

```bash
curl "http://localhost:8000/jobs/{job_id}"
```
