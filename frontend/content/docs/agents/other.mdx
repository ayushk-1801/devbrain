These clients don't have dedicated pages but work with DevBrain using standard MCP configuration.

## Windsurf

Windsurf is an AI-native IDE from Codeium. Add DevBrain via the IDE settings:

1. Open **Windsurf Settings** → **MCP Servers**
2. Click **Add Server**
3. Paste the configuration:

```json tab="Docker"
{
  "command": "docker",
  "args": [
    "run", "-i", "--rm",
    "-e", "DEVBRAIN_API_URL=http://host.docker.internal:8000",
    "devbrain",
    "python", "-m", "backend.mcp_server"
  ]
}
```

```json tab="Python"
{
  "command": "python",
  "args": ["-m", "backend.mcp_server"],
  "env": {
    "DEVBRAIN_API_URL": "http://localhost:8000"
  }
}
```

## Continue.dev

Continue is an open-source AI code assistant for VS Code and JetBrains. Add to `~/.continue/config.json`:

```json tab="Docker"
{
  "experimental": {
    "mcpServers": {
      "devbrain": {
        "command": "docker",
        "args": [
          "run", "-i", "--rm",
          "-e", "DEVBRAIN_API_URL=http://host.docker.internal:8000",
```

## Genkit

Google's Genkit framework supports stdio MCP tool providers. Register DevBrain as a tool provider when configuring your Genkit flow:

```ts
import { mcpToolProvider } from '@genkit-ai/mcp';

const provider = mcpToolProvider({
  command: 'python',
  args: ['-m', 'backend.mcp_server'],
  env: { DEVBRAIN_API_URL: 'http://localhost:8000' }
});
```

For a generic MCP configuration template that works with any client, see the [Generic Config](./generic) page.
