---
title: Issues
description: GitHub issue management endpoints.
---

DevBrain provides endpoints to create, read, update, and manage GitHub issues directly.

## Create an Issue

```http
POST /issues
```

```json
{
  "repo": "owner/repo-name",
  "title": "Bug: login fails on Safari",
  "body": "Steps to reproduce...",
  "labels": ["bug", "frontend"],
  "assignees": ["alice"]
}
```

## List Issues

```http
GET /issues?repo=owner/repo-name&state=open
```

## Update an Issue

```http
PATCH /issues/{number}
```

```json
{
  "title": "Updated title",
  "labels": ["bug", "critical"]
}
```

## Close / Reopen

```http
POST /issues/{number}/close
POST /issues/{number}/reopen
```

<Callout type="info" title="Authentication">
Requires `X-GitHub-User-Token` header (your own GitHub PAT) for write actions.
See the [agent setup docs](../agents/claude-code) for details.
</Callout>
