DevBrain is built on a **six-stage ECL pipeline** powered by Cognee's hybrid graph-vector memory layer.

## High-Level Flow

```mermaid
graph TD
    A[GitHub Repository] -->|Commits, PRs, ADRs, Code Files| B(DevBrain Ingestion Server)
    B -->|Structures context into memory payloads| C(Cognee Memory Engine)
    C -->|1. Classify| D[Vector Store / LanceDB]
    C -->|2. Check Permissions| E[Graph Store / Kuzu]
    C -->|3. Extract Chunks| F[Relational / SQLite]
    C -->|4. LLM Extract Entities| D
    C -->|5. Generate Summaries| E
    C -->|6. Embed & Commit| F
```

## Cognee 6-Stage ECL Pipeline

1. **Classify** - categorize incoming documents by type
2. **Check permissions** - enforce access controls
3. **Extract chunks** - split documents into processable segments
4. **LLM extract** - identify entities and relationships from text
5. **Generate summaries** - create concise document summaries
6. **Embed & commit** - vectorize and store in all three stores

## Three-Store Architecture

| Store | Technology | Purpose |
|-------|-----------|---------|
| Graph | Kuzu / Neo4j | AST nodes, call graphs, PR/commit connections |
| Vector | LanceDB | Semantic search and fuzzy concept matching |
| Relational | SQLite | System state and mappings |

## Services

| Service | Role |
|---------|------|
| FastAPI | REST API + webhook receiver |
| ARQ Worker | Background Cognee ingestion jobs |
| Redis | Job queue + changelog/profile state |
| Cognee MCP | Agent interface (Claude Code, Cursor) |
