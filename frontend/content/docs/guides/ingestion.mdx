DevBrain ingests five data sources into Cognee's hybrid graph-vector memory.

## Data Source Summary

<TypeTable
  type={{
    Commits: { type: "Source", description: "Message, diff summary, author, files changed" },
    "Pull Requests": { type: "Source", description: "Title, description, reviews, discussions, approvals" },
    ADRs: { type: "Source", description: "Architecture Decision Records from docs/decisions/" },
    "Codebase (AST)": { type: "Source", description: "Functions, classes, imports, call graphs via `cognee[codegraph]`" },
    "Releases & Tags": { type: "Source", description: "Release notes, git tags, deployment events" },
  }}
/>

---

## Commits

<Callout type="info" title="Trigger">
Ingested on every `push` webhook event.
</Callout>

| Nodes | Edges |
|-------|-------|
| `Commit`, `Developer`, `File` | `Commit → MODIFIED → File`, `Commit → AUTHORED_BY → Developer` |

## Pull Requests

<Callout type="info" title="Trigger">
Ingested on every `pull_request` webhook event.
</Callout>

| Nodes | Edges |
|-------|-------|
| `PullRequest`, `Developer`, `File`, `Issue`, `Review` | `PR → CHANGED → File`, `PR → MOTIVATED_BY → Issue`, `PR → APPROVED_BY → Developer` |

## Architecture Decision Records (ADRs)

<Callout type="info" title="Scan Locations">
Scanned from `/docs/decisions/`, `/adr/`, `/docs/adr/`, `/.decisions/`.
</Callout>

| Nodes | Edges |
|-------|-------|
| `ADR`, `Module` | `ADR → JUSTIFIES → Module`, `ADR → SUPERSEDES → ADR` |

## Codebase (AST)

<Callout type="info" title="Languages">
Python, JS, TS, Go, Rust, Java, C#, PHP, Ruby via `cognee[codegraph]`.
</Callout>

| Nodes | Edges |
|-------|-------|
| `Function`, `Class`, `Module` | `Function → CALLS → Function`, `Class → INHERITS → Class`, `Module → DEPENDS_ON → Module` |

## Releases & Tags

<Callout type="info" title="Trigger">
Ingested on release events.
</Callout>

| Nodes | Edges |
|-------|-------|
| `Release`, `Tag`, `Deployment` | `Release → TAGGED_BY → Tag`, `Deployment → DEPLOYED_FROM → Release` |

---

## Ingestion Modes

<Callout type="tip" title="Incremental by default">
Webhook events trigger incremental ingestion - only changed files are processed. Use `POST /ingest` for an initial full historical sync.
</Callout>

| Mode | When | Scope |
|------|------|-------|
| Incremental | Webhook events | Only changed files |
| Full sync | `POST /ingest` | Entire repo history |
