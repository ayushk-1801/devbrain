# Getting Started & Setup

DevBrain can be run using either Docker Compose (recommended) or locally with a Python virtual environment.

## Prerequisites

Before running DevBrain, make sure you have:
- **Python 3.11+** (if running locally without Docker)
- **Docker & Docker Compose** (if using the recommended setup)
- **Redis** running locally (if running without Docker)
- An **OpenAI API Key** (or another provider supported by Cognee)
- A **GitHub Personal Access Token (PAT)** with repo read and webhook access

---

## Environment Variables

Copy the `.env.example` file to `.env` in the project root:

```bash
cp .env.example .env
```

And configure the following variables:

```bash
OPENAI_API_KEY=sk-...               # Your OpenAI API key
GITHUB_TOKEN=ghp_...                # Your GitHub PAT
GITHUB_REPO=owner/repo-name         # Default repository to ingest
GITHUB_WEBHOOK_SECRET=your_secret   # For webhook verification
COGNEE_DATA_DIR=./.cognee/data
COGNEE_SYSTEM_DIR=./.cognee/system
```

---

## Option A: Docker (Recommended)

To build and start all DevBrain services:

```bash
docker-compose up --build
```

This will run the following services:
- **`redis`**: Task queue & subscription/profile storage
- **`devbrain-api`**: FastAPI API server running on port `8000`
- **`devbrain-worker`**: ARQ worker executing Cognee ingestion jobs in the background
- **`devbrain-mcp`**: Cognee MCP server for integration with Claude Code / Cursor

---

## Option B: Local Setup (No Docker)

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Start Redis
Make sure Redis is running on `localhost:6379`.
- macOS: `brew services start redis`
- Linux: `sudo systemctl start redis`

### 3. Run the API Server
```bash
uvicorn backend.main:app --reload --port 8000
```

### 4. Run the Worker
In a separate terminal tab:
```bash
python -m arq backend.worker.WorkerSettings
```
Both API and worker processes must run concurrently.
